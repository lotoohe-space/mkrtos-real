STM32F2xx Standard Peripherals Library Examples update History


V1.1.0 / 13-April-2012

Main Changes

  • All source files: license disclaimer text update and add link to the License file on ST Internet.
  • All examples: update done to be in line with latest version of STM32_EVAL architecture; include header file "stm322xg_eval.h" instead of "stm32_eval.h"
  • ADC examples 
    • ADC configuration: add initialization of the ADC_ExternalTrigConv member of the ADC_InitStructure.
    • DualADC_Interleaved_DMAmode3, DualADC_RegulSimu_DMAmode1 and TripleADC_Interleaved_DMAmode2 examples:
      • Remove call to ADC_DMACmd() function (already done by ADC_MultiModeDMARequestAfterLastTransferCmd() function).
  • CAN examples 
    • Remove CAN_StructInit() function and replace it by an explicit data initialization.
    • Remove reference to CAN_Baudrate_Calculator.xls file inside the readme.txt.
  • CortexM3/MPU example
    • Add the following defines 
#define MPU_RASR_XN_Pos                          28  
#define MPU_RASR_XN_Msk                          (1UL << MPU_RASR_XN_Pos)                       
#define MPU_RASR_AP_Pos                          24  
#define MPU_RASR_AP_Msk                          (7UL << MPU_RASR_AP_Pos)
  • CRYP examples 
    • AES_ECBmode_DMA 
      • Update Crypto and DMA configuration sequence: DMA enabled before Crypto to avoid flag error (FIFO error) generation during DMA transfer.
      • Replace "while (DMA_GetFlagStatus(DMA2_Stream6, DMA_FLAG_TCIF5) == RESET);" by "while (DMA_GetFlagStatus(DMA2_Stream5, DMA_FLAG_TCIF5) == RESET);".
    • TDES_ECBmode_DMA
      • Update Crypto and DMA configuration sequence: DMA enabled before Crypto to avoid flag error (FIFO error) generation during DMA transfer.
      • Replace "while (DMA_GetFlagStatus(DMA2_Stream6, DMA_FLAG_TCIF5) == RESET);" by "while (DMA_GetFlagStatus(DMA2_Stream5, DMA_FLAG_TCIF5) == RESET);".
  • DAC example
    • Update the DMA Stream number for each DAC Channel : DMA1 stream 5 configured with DAC Channel1 and DMA1 stream 6 with DAC Channel2.
  • DMA example
    • In stm32f2xx_it.c : replace DAM_STREAM_IRQHANDLER macro name by DMA_STREAM_IRQHANDLER.
  • DCMI example
    • Add driver for new camera OV2640 and example enhancement.
    • Change the folder name from "OV9655_Camera" to "Camera".
  • IWDG Example
    • Change EXTI interrupt Line from Line 0 to Line 15.
    • Change Invalid Address from 0x70000000 to 0xA0001000.
  • PWR examples
    • CurrentConsumption example
      • Example enhancement: when system was resumed from STANDBY mode LED1 start toggling (same behavior as sleep or stop mode resume). 
      • readme file: add more details about jumper settings
    • Update readme file for all examples : add the following note "This example can not be used in DEBUG mode, this is due to the fact that the Cortex™-M3 core is no longer clocked during low power mode so debugging features are disabled".
  • SDIO example 
    • In stm32f2xx_it.c : add SD_SDIO_DMA_IRQHANDLER() interrupt function which call SD_ProcessDMAIRQ() function at each DMA end of transfer.
    • In main.c : add the following NVIC configuration
      NVIC_InitStructure.NVIC_IRQChannel = SD_SDIO_DMA_IRQn;
      NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 1;
      NVIC_Init(&NVIC_InitStructure);  
  • SPI examples 
    • SPI_DataExchangeInterrupt : Source Code modularity enhancement. 
  • TIM examples
    • ExtTrigger_Synchro example
      •  In main.c : replace comment "GPIOA Configuration: PA.09(TIM1 CH2)" by "GPIOE Configuration: PE.11(TIM1 CH2)" 
      • In readme.txt : replace "TIM3 CH1 (PB.04)" by "TIM3 CH1 (PC.06)"
    • OCInactive example
      • Add the GPIO configuration en alternate mode for PA0/PA1/PA2/PA3 which correspond respectively to TIM2_CH1/CH2/CH3/CH4.
    • In all examples clarify TIM frequency computation formula.
  • USART examples 
    • DataExchangeDMA : Source Code modularity enhancement.
  • WWDG Example
    • Change EXTI interrupt Line from Line 0 to Line 15
    • Change Invalid Address from 0x70000000 to 0xA0001000
  • I2C Examples: add readme file to refer to the I2C CPAL Library
  • RTC examples: add readme file to refer to the "AN3371 Using the STM32 hardware real-time clock (RTC)"
